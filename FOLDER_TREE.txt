DATABASE MODELS
===============

User
  - id, email, name, role (ADMIN/CUSTOMER)
  - clerkId (Clerk integration)
  - orders (relation)

Product
  - id, name, slug, description
  - price, stock, images[]
  - attributes (JSON for specs)
  - featured, categoryId
  - category, orderItems (relations)

Category
  - id, name, slug
  - description, image
  - products (relation)

Order
  - id, userId, totalAmount
  - status (PENDING/PAID/SHIPPED/DELIVERED)
  - paymentMethod (PAYPAL/KCB_BUNI)
  - phoneNumber, transactionReference
  - shippingAddress (JSON)
  - user, orderItems (relations)

OrderItem
  - id, orderId, productId
  - quantity, price
  - order, product (relations)

SCRIPTS
=======

Development:
  pnpm dev              # Start dev server (localhost:3000)
  pnpm build            # Build for production
  pnpm start            # Start production server
  pnpm lint             # Run ESLint

Database:
  pnpm prisma:generate  # Generate Prisma client
  pnpm prisma:migrate   # Run migrations
  pnpm prisma:studio    # Open Prisma Studio GUI
  pnpm prisma:seed      # Seed database
